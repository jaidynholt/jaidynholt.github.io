Robot Arm OpenGL Project

2025


An interactive robot arm with a joint which moves and rotates on a XZ plane and shoots a projectile, teleporting to where it lands.
This is my first project made with OpenGL and focuses on affine hierarchical transformations, drawing, and lighting with shaders.
See the full project description and code on GitHub!

See & build the complete project on GitHub:
https://github.com/jaidynholt/OpenGLProject1_RobotArm

Project & Code Walkthrough

Features

Interactivity

The camera is fixed looking at the origin and moves around the origin in a sphere:

  • Select camera: press c key
  • Move camera: press ← → ↑ ↓ keys

The robot can move and rotate on the XZ plane:

  • Select robot base to move: press t key
  • Move robot base: press ← → ↑ ↓ keys
  • Select robot base to rotate: L mouse click on it or press r key
  • Rotate robot base: press ← → keys

The two arm segments can rotate:

  • Select arm segment 1: L mouse click on it or press 1 key
  • Select arm segment 2: L mouse click on it or press 2 key
  • Rotate arm segment: press ↑ ↓ keys

The robot can shoot a projectile out of its arm (with force equal to its length) and teleport to where it lands:

  • Shoot projectile: press s key


Hierarchical Transformations

Each MeshObject contains its own model matrix (4x4 matrix) and translate (vector 3) and rotate (quaternion). Each can optionally contain MeshObject children. When a transform is applied to a MeshObject, its translate or rotate is updated. Each time its world model matrix is needed (such as for drawing), its translate and rotate and all its predecessors' translates and rotates are applied to its model matrix.



Drawing

Each MeshObject contains its vertex array object (VAO) and vertex buffer objects (VBO) for its vertices, uvs, and normals. Its model is loaded from an OBJ file and its data is sent to the buffers for the GPU to render.
The models for the project were made and exported in Maya.



Lighting

There are two lights in the scene: red and blue. The lights are represented as positions with values for their various components which the shader uses to approximate raycast light:

  • Diffuse light: Light that hits an object and disperses, effectively coloring the object in areas reachable from the light's position and attenuating with distance. Its strength is determined using the angle between the incident light vector and the normal, where the smaller the angle, the more light is seen.
  • Specular light: Light that hits an object and bounces with minimal dispersion, creating specular highlight at the incident location. Its strength is determined using the angle between the incident light vector and the vector to the camera, where the smaller the angle, the brighter the highlight.
  • Ambient light: Light that colors the entire object, regardless of the light's source. This is implemented as a color in the shader, serving as an approximation of raycast light where light bounces around many surfaces and is seen almost everywhere.
